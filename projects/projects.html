Human Activity Recognition in Electric Line Workers from Single Wrist Accelerometer

Github Link

We recruited 44 subjects to perform 10 activities to simulate electric line worker's daily tasks. The tasks included:

  1. Sitting
  2. Stainding
  3. Walking
  4. Hoisting a Bucket
  5. Lifting & Lowering a Box
  6. Pushing a Cart
  7. Typing on a Computer
  8. Climbing up and down a Ladder
  9. Working in an Electric Panel
  10. Inserting a Screw at overhead Level
The subjects were equipped with GE proprietary wearables and the sensor data were transfered to the cloud during the sessions. I used the signals to create features in 3 domains:
  • Time
  • Frequency
  • Time-Frequency
and performed machine learning methods to classify the undertaken tasks. I used the Scikit-Learn and TensorFlow libraries in Python to do that.

Also, to assure similar performance of the classification when launched, I used the LIME method to find the relevance of the features that contribute most to the prediction by reconstructing the signals from the frequency and time-frequency features.

This work is under review in PLOS One journal.

Gait Fatigue Analysis form Ankle Sensor using Hotelling's T2 Control Charts

Code and R-Markdown

We recruited 12 subjects to perform manual material handling tasks for 3 hours. Using the the IMU signals that was attached to their right ankle we tried to dtect fatigues. We developed a statistical process control (SPC) approach to construct a control chart.

We grouped the gait cycles during continuous walking periods into subgroups and assigned the first few subgroups to baseline and designed a 3-phase SPC approach:

  1. Phase 0: to remove outliers & find the within subgroup covariance
  2. Phase 1: to find the upper control limit using a bootstrapping technique
  3. Phase 2: the online data to be monitored

The subgroups whose T2 statistic exceeded the control limit created an out-of-control signal and considered fatigued.

Aside from fatigue detection this approach can be used for detecting the gait deterioration in certain patients as well as the gait improvement in the rehabilitation process. This can be done by modifying the subgrouping period to days or weeks.

Gait Fatigue Analysis form Ankle Sensor using Functional Depth and EWMA Control Charts

Code and R-Markdown

Using the gait data from the previous study, we tried to use a cumulative SPC approach (EWMA) to detect fatigue.

After segmenting the gait cycles from the IMU signals, we assigned the first 500 cycles to the baseline and calculated the depth of the remaining gait profiles with respect to the baseline. We used 2 different functional depth measures:

  • Mode Depth
  • Multivariate Functional Halfspace Depth (MFHD)
calculation of MFHD depth was computationally expensive and therefore, we performed the calculations in parallel on 40 nodes on Ohio Super Computing (OSC) center.

After calculating the depth measures, we reassigned the values to their original subgroups and constructed the EWMA control chart by considering the 1st 10 subgroups as the in-control data. The gaits with EWMA statistic larger than the control limit are considered fatigued.

This is an ongoing project.

Fairness in Machine Learning

COMPAS is a machine learning (ML) algorithm that is widely used in the US judiciary system to help the judge to decide to make decisions about the bail/parole eligibility by prdicting how likely it is for the person to commit a crime in the future. While race is not one of the features used by COMPASS for prediction, an independent non-profit news organization, evaluated the algorithm on certain data and claimed when the algorithm was incorrect it tended to skew very differently for different races.

Along these lines we tried to optimize the post-processing evaluation methods to improve the fairness in 3 ML models:

  • Linear Support Vector Regressor
  • Feed-forward Neural Network
  • Naive Bayes Classifier
The 3 models were trained on a data set from the Broward county. We invetigated 5 potential post-processing methods that enforce different constraints in attempts to reflect different measures of fairnessWe in ML. The evaluation methods included:
  1. Maximum profit/Maximum Accuracy
  2. Single Threshold
  3. Predictive parity
  4. Demographic parity
  5. Equal Opportunity
Using each of these approaches the accuracy and other performance metrics were compromised in order to achieve higher fairness using different fairness metrics. I have included a sample output presenting the FPR, FNR, TPR, TNR, and accuracy for each race as well as the overal accuracy and cost. The best fairness metric was selected and justified based on our discretion.

Coding 5 Clustering & 4 Classification Algorithms from scratch in Python

Github Link

I implemented 5 clustering algorithms in Python in order to find clusters of genes that exhibit similar expression profiles without the use of existing libraries. The 5 Clustering algorithms included:

  1. K-means
  2. Hierarchical Agglomerative
  3. Density-based
  4. Mixture Model
  5. Spectral Clustering
Using the true clusters I validated my results using external index (Rand Index and Jaccard Coefficient). I also visualize data sets and clustering results by Principal omponent Analysis (PCA) (also implemented by me).

I also implemented 4 classification algorithms in Python without the use of existing libraries wich included:

  1. Nearest Neighbor (k-NN)
  2. Decision Tree
  3. Naive Bayes
  4. Random Forest
and using a 10-fold cross validation I evaluated the performance of each method on two datasets in terms of accuracy, precision, recall, and F-1 score.





Analysis of Back & Shoulder Posture in Palletizing Task

In a project that was funded by the Gore company we investigated the effect of fatigue on the back and shoulder postures. We recruited 9 participants and attached reflective markers at the land marks on their upper body as shown in the top figure in the right panel. Thesubjects perfomed palletinzing task for 3 sessions of 10 minutes duration. The marker data were collected using Vicon motion capture system.

I used the marker coordinates to calculate the Euler angles at the soulder based on the ISB standards (flexion-extension, adduction-abduction, and the axial rotation). I also calculated the back angle with respect to the global vertical. We are investigated the change in the back and shoulder flexion-extension in terms of the angles, their timing, and also the coordination between back and shoulder. The angles and results are visualized in the two bottom figures in the right panel.

In this project, the participants were also equipped with commercial wearable sensors including Xsens, Dorsavi, and the ADI wearable shirt. We validated the upper body angles calculated by each of these sensors against the Vicon data.

This is an ongoing project.
